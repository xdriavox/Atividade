1. O que é GitHub e qual é a sua principal funcionalidade?

GitHub é uma plataforma de hospedagem de código que permite que desenvolvedores colaborem em projetos de software. Sua principal funcionalidade é gerenciar repositórios Git, facilitando o versionamento de código e a colaboração entre equipes.

2. Quais são as diferenças entre Git e GitHub?

O Git é um sistema de controle de versão distribuído que permite o rastreamento de alterações no código-fonte durante o desenvolvimento. Já o GitHub é uma plataforma que hospeda repositórios Git na nuvem, oferecendo recursos adicionais como pull requests, issues, e integração contínua.

3. Como criar uma conta no GitHub?

Para criar uma conta no GitHub, acesse o site github.com e siga as instruções de cadastro. Você precisará fornecer um endereço de e-mail válido, criar um nome de usuário e uma senha.

4. Quais são os primeiros passos após a criação da conta no GitHub?

Após criar sua conta, você pode configurar seu perfil, criar seu primeiro repositório e conectar o GitHub ao seu computador usando Git.

5. Como configurar o Git no seu computador local para se conectar ao GitHub?

Instale o Git no seu computador e configure suas credenciais de usuário utilizando os seguintes comandos:

                git config --global user.name "Seu Nome"
                git config --global user.email "seuemail@example.com"

6. O que é um repositório no GitHub?

Um repositório no GitHub é um local onde todo o código-fonte de um projeto, junto com o histórico de mudanças, é armazenado. Ele pode ser público ou privado.

7. Como criar um novo repositório no GitHub?

Após fazer login no GitHub, vá para a página de repositórios e clique em "New Repository". Dê um nome ao seu repositório, escolha se será público ou privado, e clique em "Create Repository".

8. Qual é a diferença entre um repositório público e um repositório privado?

Repositórios públicos são visíveis para qualquer pessoa na internet, enquanto repositórios privados só podem ser acessados por pessoas autorizadas.

9. O que significa clonar um repositório e como isso é feito?

Clonar um repositório significa copiar um repositório Git de um servidor remoto (como o GitHub) para o seu computador local. Isso é feito com o comando git clone URL.

10. Qual é a utilidade de clonar um repositório?

Clonar um repositório permite que você trabalhe em um projeto localmente, com todo o histórico de commits e arquivos disponíveis em seu computador.

11. Quais são os principais comandos do Git que são usados com frequência?
  • git init: Inicializa um novo repositório Git.
  • git clone: Clona um repositório existente.
  • git add: Adiciona arquivos ao índice (staging area).
  • git commit: Grava as mudanças no repositório local.
  • git push: Envia as mudanças para o repositório remoto.
  • git pull: Puxa as mudanças do repositório remoto para o local.

12. Como fazer commits e qual a importância de uma mensagem de commit clara e descritiva?

Fazer commits com mensagens claras e descritivas é importante para o entendimento do histórico de mudanças em um projeto. Use o comando git commit -m "Sua mensagem de commit" para realizar um commit.

13. O que são branches e como criar uma nova branch no GitHub?

Branches são ramificações do repositório principal, usadas para desenvolver funcionalidades isoladamente. Crie uma nova branch com o comando git branch nome-da-branch e alterne para ela com git checkout nome-da-branch.

14. Como fazer merge de branches e quais são as boas práticas para isso?

Para fazer merge de branches, use o comando git merge nome-da-branch na branch que você deseja unir. É uma boa prática revisar os conflitos antes de fazer o merge.

15. Como fazer um fork de um repositório?

Forkar um repositório significa criar uma cópia dele na sua conta do GitHub. Isso permite que você faça alterações sem afetar o repositório original. Use a opção "Fork" no GitHub para isso.

16. O que é um pull request e como criar um pull request?

Um pull request é uma solicitação para que as mudanças feitas em um fork ou branch sejam integradas ao repositório principal. Crie um pull request no GitHub após fazer push das suas alterações.

17. Como revisar e aceitar um pull request?

O revisor deve analisar o código, testar as mudanças e, se tudo estiver correto, aceitar o pull request para que as mudanças sejam mescladas no repositório principal.

18. O que são issues e como abrir uma issue no GitHub?

Issues são relatórios de problemas ou sugestões de melhorias em um projeto. Para abrir uma issue, vá à aba "Issues" de um repositório no GitHub e clique em "New Issue".

19. Como usar o sistema de projetos do GitHub para gerenciar tarefas?

O GitHub oferece um sistema de gerenciamento de projetos onde você pode criar quadros Kanban para organizar e acompanhar o progresso das tarefas. Acesse a aba "Projects" e crie um novo projeto para começar.
